## For future Claude

This is a source page about JLL (Jones Lang LaSalle) Portugal Residential. It documents the dlt-driven SCD2 ingest with per-entity heartbeat sidecars, the deliberate omission of plots ingestion (vui-parameter dependency), and the schema-contract hard-fail policy. Read this page before editing [pipelines/portals/jll/jll_dlt_dag.py](../../pipelines/portals/jll/jll_dlt_dag.py) or its source module.

## Source

- **Official name**: JLL Residential PT (Jones Lang LaSalle Portugal)
- **Owner**: Jones Lang LaSalle (private company; commercial real-estate brokerage with residential vertical)
- **Protocol**: dlt + REST JSON — Supabase-style endpoint behind a JLL-controlled imoguia.com proxy
- **Base endpoint**: `https://vcs.imoguia.com/jll/api`
- **License**: proprietary; data accessed via JLL's public listing endpoint
- **Schedule**: weekly Thursdays 06:00 UTC (`0 6 * * 4`) — matches JLL's upstream refresh cadence
- **Source pattern**: dlt with SCD2 write-disposition, per [[scd2-row-hash]]

## Schema

Two SCD2 fact tables + per-entity heartbeat sidecars (per [[heartbeat-sidecar]]):

- `bronze_listings.jll_listings` — SCD2 with `_dlt_id`, `_dlt_valid_from`, `_dlt_valid_to`, `_dlt_sn` (sequence number), all listing attributes
- `bronze_listings.jll_developments` — SCD2 with same pattern; new-build developments
- `bronze_listings.heartbeat_listings` / `heartbeat_developments` — sidecar UPSERT-only tables tracking `last_seen_date` per entity. Lets dbt staging distinguish "listing changed" from "listing not in this scrape."

## Quirks

- **Plots NOT ingested** (deliberate): the `/v1/Properties` endpoint that returns plot inventory requires a per-visitor `vui` parameter generated by a real browser. Reverse-engineering it for a headless ingest is fragile; PT plot inventory is already covered by [[idealista]], [[remax]], and [[zome]]. JLL plots add no unique signal worth the effort.
- **Schema-contract hard-fail**: the DAG calls `dlt`'s schema-contract enforcement; on contract violation (new field appears, type changes, required field disappears), the load_facts task hard-fails and an `on_failure_callback` alert fires. Bronze never accepts unanticipated schema drift silently — adheres to [[bronze-permissive]] (permissive on values, strict on schema shape).
- **Validation bands**: post-load checks assert `listings_current` ∈ [1000, 30000] and `developments_current` ∈ [50, 500]. Outside bands → fail. Catches both upstream collapses (empty response) and runaway duplicates.
- **Audit copy**: best-effort raw JSON copy to MinIO under `audit/jll/<date>/` runs as a separate task; failure is non-blocking (load_facts continues on its own dlt-managed fetch).
- **Three-task DAG topology**: `audit_to_minio` (best-effort) → `load_facts` (hard-fail SCD2 merge) → `validate_facts` (band checks).
- **Origin agency/agent fields collapsed**: per recent refactor (commit `5906fa1`), nested agency/agent objects collapsed into single JSONB columns to match dlt's flatten-or-jsonb resolution; staging models unpack as needed.
